# semantic_check_protocol.md

# 经管论文智检 Skill：语义判断执行协议

## 1. 文件作用

本文件指导 Claude Code 如何对论文中的自然语言内容进行语义判断。规则库说明检查什么，本文件说明如何判断。

适用于研究问题是否明确、文献综述是否罗列化、理论链条是否贯通、数据变量口径是否清楚、方法模型是否与数据类型匹配、结果解释是否超出表格支持、中英文摘要是否一致、结论建议是否对应前文分析等任务。

## 2. 总体判断原则

先读上下文，再下判断。先找证据，再生成问题。不用单个关键词机械判定。不把写法不完美直接判为严重错误。不把本科论文按硕博论文要求审查。不把未使用高级模型判为问题。无法稳定判断时标记“需人工确认”。输出建议时给出可执行修改方向。

## 3. 基本流程

读取 paper_profile，定位相关章节，提取证据片段，判断是否满足规则触发条件，判断证据强度，确定状态与等级，写入结构化 issue、pass_item、not_applicable_item 或 manual_confirmation_item。

不得跳过证据片段直接生成结论。

## 4. 研究问题明确性判断

判断论文是否清楚回答本文研究什么对象、围绕什么核心关系或问题、为什么值得研究、后文数据案例或理论分析是否围绕该问题展开。

可判定为通过的情况包括题目体现研究对象或核心变量，摘要说明研究目标，引言末段明确提出研究问题或研究内容，后文章节围绕该问题展开，研究问题与数据案例或理论分析对应。

可判定为问题的情况包括引言主要是政策背景或行业背景但没有明确提出问题，摘要只写研究主题没有说明具体研究对象，研究内容只是宽泛方向不是可回答的问题，后文数据模型或案例分析与前文研究问题不对应。

问题通常列为黄色；若研究问题缺失导致全文结构失焦，可提升为红色，但必须有强证据。

## 5. 文献综述罗列化判断

判断文献综述是否只是堆叠已有研究，还是形成了分类、比较、归纳和述评。

罗列化典型信号包括连续使用某某认为、某某指出、某某研究发现，缺少按主题方法变量结论或研究对象分类，缺少对已有研究差异的比较，缺少已有研究不足和本文切入点，文献综述结束后直接进入研究设计。

不应误判的情况包括论文已有分类标题只是评述略短，课程论文篇幅较短，案例分析论文只引用少量案例相关文献，理论综述型论文本身以文献梳理为主但结构清楚。

## 6. 理论链条贯通性判断

判断理论基础、机制分析、研究假设、变量设计和结论解释之间是否有对应关系。

贯通表现包括理论部分解释核心变量关系，研究假设来自文献或理论分析，变量选择对应理论概念，实证结果解释回到理论机制，结论没有脱离前文理论与结果。

脱节表现包括理论部分出现的概念后文没有使用，变量设置没有理论依据，研究假设提出突然，机制分析只写概念没有连接 X、M、Y，结论使用理论概念但前文没有解释。

理论链条问题通常列为黄色。若理论完全无法支撑核心模型或变量设计，可列为红色，但必须有强证据。

## 7. 数据、变量与口径清晰性判断

判断论文的数据、样本、变量和指标是否能被读者复核。

清晰表现包括说明数据来源、样本期间、样本对象、样本筛选规则，变量定义表包含名称符号含义公式来源，指标体系说明正负向属性、标准化方法和权重计算。

不清晰表现包括只写来源于统计年鉴或数据库，未说明具体数据库年份指标口径，只写最终样本没有剔除规则，核心变量只有名称和符号，综合指数缺少标准化权重或综合得分计算说明，摘要正文表格结论中的样本口径不一致。

数据来源不可追溯、核心变量定义缺失、口径前后不一致通常可列为红色。样本筛选说明不够细、变量表排版不完整通常列为黄色。

## 8. 方法模型与数据类型匹配判断

Claude Code 不负责替导师决定最佳模型，只检查明显不匹配或说明不足的情况。

可判定为问题的情况包括面板数据但完全未说明面板模型或固定效应，二元因变量却直接按普通连续变量解释，使用 DID 但未定义处理组对照组和政策时点，使用熵值法但未说明指标体系和权重计算，使用中介效应但未说明中介变量理论依据，使用回归模型但没有模型方程或变量说明。

不应判定为问题的情况包括未使用 DID、未使用 IV、未使用 PSM、未使用高级稳健性检验、本科论文没有复杂识别策略、案例论文没有回归模型。这些应进入不适用项，而不是问题项。

## 9. 结果解释与表格支持关系判断

判断正文对结果的解释是否被表格、模型或案例材料支持。

可判定为问题的情况包括正文说显著但表格中不显著，正文解释方向与系数方向相反，结论提出表格中没有检验的机制，普通回归结果被写成强因果结论，案例论文结论超出案例材料支持，问卷论文结论超出样本范围。

如果表格无法解析，应标记“回归表无法稳定解析，需人工确认正文解释是否与系数方向和显著性一致”。不得在表格无法解析时直接判定正文解释错误。

## 9ter. 语言强度与证据强度对齐判断（超范围结论）

本节处理一类高频、隐蔽的问题：论文中语言强度（statistical / causal 强度）超出实际证据强度，属于结论超出识别策略或证据范围。典型模式是摘要或结论使用统计学专有词（如“显著”“双重抑制”“证明”“因果”），但对应维度并未做实证；或使用具体比较词（如“更明显”“更突出”“更强”）但未做分组/异质性回归。

判断步骤：

1. **提取语言强度关键词**。摘要、副标题、结论前段中检查是否出现以下三类词：
   - 统计学专有词：显著、显著为正/负、通过检验、拒绝原假设、p 小于 0.05；
   - 因果强度词：证明、验证了、导致、造成、决定、因果效应；
   - 具体比较词：更明显、更突出、更强、优于、大于、排序为 X＞Y＞Z。
2. **回溯对应实证证据**。在实证章节中查找是否存在支撑该语言强度的回归表、系数、显著性星号、分组回归、交互项或稳健性检验。
3. **判断对齐关系**。
   - 语言强度 ≤ 证据强度 → 通过（对齐）；
   - 语言强度 > 证据强度 且证据可核对 → 判为红色，进入 issue_writing_protocol 第 15bis 节双路径建议流程；
   - 语言强度 > 证据强度 但证据涉及表格或公式为图片无法核对 → 转为需人工确认。

判级要点：

- 不得因“作者可能只是想做辅助分析”“作者在结论章末尾做了免责声明”就直接把红色降为黄色。免责声明的存在只影响修改建议的写法（可优先推荐语言修订路径），不改变判级。
- 免责声明必须在摘要就出现才能视为“语言与证据对齐”。若免责声明只出现在结论章末尾而摘要仍使用强语言，仍然判红。
- 需要区分两类不同情况：一是作者知道边界但表达不严谨（免责声明存在但摘要透支），二是作者把文献综述里别人的发现当作自己的发现（无免责声明且完全缺乏实证）。前者是表达问题，后者是学术规范问题，两者均判红但双路径建议的措辞不同。

## 9bis. 强制交叉核对项（必须逐项执行）

本节将容易被漏检的一致性核对从“建议”升级为“必须执行”。语义判断阶段必须完整走完以下每一项，不得跳过；每项都要在报告的对应诊断域留下痕迹（判为问题、通过项或需人工确认之一）。这些是不同智能体最容易产生分歧、也是导师最常挑的地方。

必须逐项执行的交叉核对：

1. **描述性统计：正文数字 vs 表格数字**。逐一比对正文叙述的每个统计量（均值、标准差、最小值、最大值、样本量）与描述性统计表中的对应数字。发现任一不一致，按 evidence_requirement.md 第 9bis 节判为红色。
2. **回归系数：正文叙述 vs 回归表**。逐一比对正文对基准回归、稳健性、内生性各列所述的系数、正负号、显著性与表中数字是否一致。
3. **异质性/分组结论：文字排序 vs 表格系数排序**。对每一组异质性或分组回归（如东中西部、功能区），核对正文/摘要/结论所述的大小排序、强弱梯度、是否显著，与表中系数逐一对照。文字与系数排序矛盾者判红。
4. **摘要—正文—结论三方对照**。将摘要的主要结论、正文实证结果、结论章节的表述三方对照，核对结论方向、机制、异质性描述是否一致。
5. **中英文摘要逐要素对照**。按研究对象、样本期间、地区范围、核心方法、主要结论、政策时期、关键词，逐要素比对中英文摘要，并检查英文摘要内部是否自相矛盾。
6. **指标数量核对**。核对指标体系表中实际可数的三级指标数量，与正文声称的数量是否一致；正文用“多项”等含糊表述而未给确切数量的，判为黄色。
7. **正文引用 vs 参考文献双向对照**。抽查正文上标/夹注引用是否都能在参考文献中找到对应条目，参考文献编号是否连续无缺漏。

执行要求：完成后须能说明以上 7 项各自的核对结论（一致 / 发现问题 / 需人工确认）。任何一项因表格或公式为图片无法核对时，转为需人工确认，并在报告中写明无法核对的具体对象，不得静默跳过。

## 10. 中英文摘要一致性判断

重点比较研究对象、样本期间、地区或行业范围、核心变量、研究方法、主要结论、政策时期、关键词翻译。

可判定为问题的情况包括中文摘要写 2011—2023 年但英文摘要写 2012—2022 年，中文摘要写长三角城市但英文摘要写中国省份，中文摘要有中介机制但英文摘要遗漏，中文摘要结论为显著正向但英文摘要表述为负向或不确定。

关键信息明显不一致可列为红色。术语翻译不统一但不影响核心信息，列为黄色或绿色。

## 11. 结论与建议对应性判断

判断结论是否回应研究问题，建议是否来自研究发现。

可判定为问题的情况包括结论只是重复摘要，结论没有对应实证结果或案例发现，结论出现前文未分析的新判断，建议套话化，建议对象不明确，建议与研究发现不对应。

建议学生使用“研究发现—对应主体—具体行动”的结构，不得替学生整段代写。

## 12. 论文片段与章节缺失处理

需要区分论文确实缺少某部分和用户上传文档可能只是论文片段。若文档没有封面题目或摘要、从中间章节开始、没有参考文献但正文明显未结束、章节编号不完整、文件名包含节选片段部分章节等，应降低缺失类问题等级，并将相关完整性问题标记为需人工确认。

## 13. 语义判断置信度

高置信度表示原文证据清楚、位置明确、上下文一致、问题与规则高度匹配、不依赖外部模板或导师偏好。

中置信度表示原文有相关证据但说明不充分，需要结合上下文判断，可能构成问题但存在解释空间。

低置信度表示原文信息不足、章节可能缺失、表格公式无法解析、是否构成问题取决于导师或学校模板。低置信度判断不得直接列为红色问题。

## 14. 反过度判断规则

不应将所有“不充分”都升级为“错误”。文献综述有分类但述评较短、样本筛选说明存在但不够细、变量定义表存在但个别变量说明略简、稳健性检验有但解释不充分、政策建议有对应对象但不够具体、英文摘要表达不够地道但核心信息一致，通常不应列为红色。

只有当问题影响论文核心逻辑、数据可复核性或结论可信度时，才可升级为红色。

## 15. 语义判断质量门禁

完成语义判断后，必须检查是否读取 paper_profile、是否定位到相关章节、是否提取原文证据、是否根据上下文判断而不是关键词机械判断、是否标记证据强度和判断置信度、是否避免硕博化、是否避免强制高级模型、是否将不确定事项转为需人工确认、是否能映射到 issue_schema、是否能写入最终 DOCX 报告、是否避免生成论文代写内容。
