# 经管论文智检 Skill：证据要求与防编造规范

## 1. 文件作用

本文件规定 Claude Code 使用本 Skill 进行论文质检时必须遵守的证据要求。所有诊断结论都必须基于用户上传 DOCX 论文中的可见内容。

本文件优先级高于具体规则库。当规则触发条件与证据要求冲突时，以证据要求为准。

## 2. 核心原则

1. 没有原文证据，不输出问题。
2. 无法稳定判断，输出“需人工确认”。
3. 不得把“未检测到”写成“学生没有做”。
4. 不得编造变量、模型、数据来源、表格、参考文献或结论。
5. 不得联网补充论文中没有呈现的内容。
6. 不得将规则库中的示例当成论文事实。
7. 不得将本科论文按硕士、博士或盲审标准强行判错。
8. 不得因为论文没有使用高级模型就判定论文存在问题。
9. 所有红色问题必须有强证据。
10. 所有需人工确认事项不得直接计入红色必改问题。

## 3. 可接受证据类型

可接受证据包括论文题目、中文摘要、英文摘要、关键词、目录、章节标题、正文段落、模型公式、变量定义表、数据来源说明、指标体系表、描述性统计表、相关性分析表、回归结果表、图表标题、图表注释、结论与政策建议、正文引用、参考文献、附录中可解析内容。

## 4. 不可接受证据类型

不可接受证据包括 Claude Code 的主观感觉、通用写作经验、规则库示例文字、其他论文中的常见问题、外部数据库真实情况、未经用户上传的学校模板、未联网核验的参考文献真伪、对学生写作过程的猜测、对导师偏好的猜测、对原始数据真实性的判断。

## 5. 证据强度

证据强度分为强、中、弱。

强证据包括原文直接呈现问题、前后两处内容明显不一致、关键章节明确缺失必要说明、表格信息与正文解释明显冲突、中英文摘要存在关键信息不一致、模型方法与数据类型明显不匹配、核心变量定义缺少公式来源或含义、数据来源表述明显不可追溯。

中证据包括原文有相关内容但说明不完整，需要结合上下文判断，问题存在较大可能但仍有解释空间。

弱证据包括原文表述模糊，内容可能位于图片、脚注或附录，表格或公式无法稳定解析，是否构成问题取决于学校模板或导师要求，论文可能只是片段。

红色问题必须有强证据。弱证据不得列为红色，应优先进入需人工确认。

## 6. 原文证据写法

每个问题必须包含 evidence 字段。证据应尽量引用论文中的短句、章节标题、表格标题或图表标题，不需要大段复制原文。若需引用前后不一致，必须同时列出两处证据。

不得写“论文写得不清楚”作为证据。

## 7. 位置要求

每个问题必须尽量给出具体位置，优先级为摘要、英文摘要、第 X 章、第 X 节、表 X、图 X、参考文献、结论、附录。无法定位时应写“位置：未能精确定位，建议人工复核相关章节”。

## 8. 未检测到的标准写法

推荐写法：论文中未检测到、当前文档未呈现、在可解析内容中未发现、建议补充、需人工确认该内容是否位于附录或图片中。

禁止写法：学生没有做、作者完全没有考虑、一定没有。

## 9. 红色问题证据门槛

红色必改问题必须满足有强证据、能定位到具体位置、影响论文核心质量、修改动作明确、不属于导师主观判断、不属于解析不稳定导致的误判。

证据为弱、是否需要该内容存在争议、论文类型不适用该规则、只是表达优化、只是高级模型建议、只是格式模板差异时，不得列为红色。

## 9bis. 红色硬伤强制清单（不得降级）

本节用于消除“同一问题在不同智能体上被判成红色或绿色”的不一致。以下情形只要在可解析内容中出现且证据确凿，**一律判为红色，不得降级为黄色或绿色，也不得以“细微偏差”“笔误”“语言表达”为由弱化**。这些属于数据可复核性与结论可信度的硬伤，是本科经管论文的核心质量红线。

强制红色清单：

1. **描述性统计数字不一致**：正文叙述的均值、标准差、最小值、最大值、样本量等，与对应描述性统计表中的数字不一致。（例：正文写“均值0.266、标准差0.089”，但表格为“0.269、0.086”。）
2. **回归系数或显著性不一致**：正文叙述的系数值、正负号、显著性水平，与对应回归表不一致。
3. **结论方向与回归表矛盾**：正文/摘要/结论对分组或异质性结果的大小排序、正负方向、强弱梯度的文字描述，与回归表实际系数不一致。（例：文字写“中部＞西部＞东部”，但表中西部系数最大。）
4. **中英文摘要关键信息不一致**：研究对象、样本期间、地区范围、核心方法、主要结论方向、政策时期等关键信息在中英文摘要之间不一致，或英文摘要内部自相矛盾。（例：中文“十五五”，英文写“14th Five-Year Plan”。）
5. **核心数据口径前后不一致**：样本期间、样本量、地区范围、核心变量口径在摘要、数据说明、表格、结论之间明显冲突。
6. **正文引用与参考文献严重不匹配**：正文明确引用的文献在参考文献中缺失，或编号体系明显错乱到无法对应（仅个别格式瑕疵不在此列，属黄色）。

判级要求：

- 上述任一情形一经确认，`level` 必须为“红色”，`evidence_strength` 必须为“强”，且 evidence 字段必须**同时列出相互矛盾的两处证据**（例如“正文段落#73:0.266”与“表格#3:0.269”）。
- 只有当相关内容位于无法解析的图片/公式对象、无法确认两处数值时，才可转为灰色需人工确认；一旦两处数值均可读且确实矛盾，不得转灰、不得降级。
- 判断“矛盾”以可解析文本为准，不得臆测；但发现可读的数字冲突时，必须判红，不得回避。

## 10. 需人工确认触发条件

表格为图片或复杂嵌入对象、模型公式无法解析、图表注释无法稳定识别、参考文献格式解析不完整、论文类型无法稳定判断、数据类型无法稳定判断、是否需要高级模型属于导师判断范围、学校格式模板未提供、原文信息可能位于附录脚注图片中、证据不足但存在潜在风险时，必须进入需人工确认。

## 11. 负证据处理规则

论文未检测到某项内容属于负证据。可以作为问题的情况包括本科经管论文通常必须呈现的基础信息未检测到、当前论文已使用某方法但未检测到该方法必要说明、论文前文声称做了某项分析但后文未检测到对应结果、摘要正文表格结论之间存在明显缺口。

不得作为问题的负证据包括论文未使用 DID、IV、PSM，案例论文未设置回归模型，课程论文未设置完整实证结构，调查数据库论文未做自编问卷信度效度检验。

## 12. 表格、公式、图片证据边界

若表格内容可读取，可以作为证据。若表格为图片、截图或复杂嵌入对象，不能直接判定表格缺少某项内容，应输出“回归表可能为图片或复杂嵌入对象，系统无法稳定识别变量名、系数、标准误和显著性星号，建议人工确认”。

若模型公式不可解析，不得直接判断模型方程错误，应输出“模型公式无法稳定解析，需人工确认变量符号、下标和固定效应项说明是否完整”。

## 13. 证据强度与等级矩阵

强证据可输出红色、黄色或绿色。中证据可输出黄色、绿色或灰色。弱证据优先输出灰色或黄色。无证据不得输出任何 issue。

## 14. 输出前证据自检

生成最终报告前逐条检查：每个问题是否有 evidence，每个问题是否有 location，evidence 是否来自用户上传论文，evidence 是否足以支撑 issue_type，红色问题是否达到强证据门槛，弱证据是否已转为需人工确认，是否存在编造变量模型或数据，是否存在把未检测到写成学生没有做，是否存在不适用规则被写成问题，是否存在高级模型强制要求。
