# evidence_requirement.md

# 经管论文智检 Skill：证据要求与防编造规范

## 1. 文件作用

本文件规定 Claude Code 使用本 Skill 进行论文质检时必须遵守的证据要求。所有诊断结论都必须基于用户上传 DOCX 论文中的可见内容。

本文件优先级高于具体规则库。当规则触发条件与证据要求冲突时，以证据要求为准。

## 2. 核心原则

1. 没有原文证据，不输出问题。
2. 无法稳定判断，输出“需人工确认”。
3. 不得把“未检测到”写成“学生没有做”。
4. 不得编造变量、模型、数据来源、表格、参考文献或结论。
5. 不得联网补充论文中没有呈现的内容。
6. 不得将规则库中的示例当成论文事实。
7. 不得将本科论文按硕士、博士或盲审标准强行判错。
8. 不得因为论文没有使用高级模型就判定论文存在问题。
9. 所有红色问题必须有强证据。
10. 所有需人工确认事项不得直接计入红色必改问题。

## 3. 可接受证据类型

可接受证据包括论文题目、中文摘要、英文摘要、关键词、目录、章节标题、正文段落、模型公式、变量定义表、数据来源说明、指标体系表、描述性统计表、相关性分析表、回归结果表、图表标题、图表注释、结论与政策建议、正文引用、参考文献、附录中可解析内容。

## 4. 不可接受证据类型

不可接受证据包括 Claude Code 的主观感觉、通用写作经验、规则库示例文字、其他论文中的常见问题、外部数据库真实情况、未经用户上传的学校模板、未联网核验的参考文献真伪、对学生写作过程的猜测、对导师偏好的猜测、对原始数据真实性的判断。

## 5. 证据强度

证据强度分为强、中、弱。

强证据包括原文直接呈现问题、前后两处内容明显不一致、关键章节明确缺失必要说明、表格信息与正文解释明显冲突、中英文摘要存在关键信息不一致、模型方法与数据类型明显不匹配、核心变量定义缺少公式来源或含义、数据来源表述明显不可追溯。

中证据包括原文有相关内容但说明不完整，需要结合上下文判断，问题存在较大可能但仍有解释空间。

弱证据包括原文表述模糊，内容可能位于图片、脚注或附录，表格或公式无法稳定解析，是否构成问题取决于学校模板或导师要求，论文可能只是片段。

红色问题必须有强证据。弱证据不得列为红色，应优先进入需人工确认。

## 6. 原文证据写法

每个问题必须包含 evidence 字段。证据应尽量引用论文中的短句、章节标题、表格标题或图表标题，不需要大段复制原文。若需引用前后不一致，必须同时列出两处证据。

不得写“论文写得不清楚”作为证据。

## 7. 位置要求

每个问题必须尽量给出具体位置，优先级为摘要、英文摘要、第 X 章、第 X 节、表 X、图 X、参考文献、结论、附录。无法定位时应写“位置：未能精确定位，建议人工复核相关章节”。

## 8. 未检测到的标准写法

推荐写法：论文中未检测到、当前文档未呈现、在可解析内容中未发现、建议补充、需人工确认该内容是否位于附录或图片中。

禁止写法：学生没有做、作者完全没有考虑、一定没有。

## 9. 红色问题证据门槛

红色必改问题必须满足有强证据、能定位到具体位置、影响论文核心质量、修改动作明确、不属于导师主观判断、不属于解析不稳定导致的误判。

证据为弱、是否需要该内容存在争议、论文类型不适用该规则、只是表达优化、只是高级模型建议、只是格式模板差异时，不得列为红色。

## 9bis. 红色硬伤强制清单（不得降级）
本节用于消除“同一问题在不同智能体上被判成红色或绿色”的不一致。以下情形只要在可解析内容中出现且证据确凿，**一律判为红色，不得降级为黄色或绿色，也不得以“细微偏差”“笔误”“语言表达”为由弱化**。这些属于数据可复核性与结论可信度的硬伤，是本科经管论文的核心质量红线。

强制红色清单：

1. **描述性统计数字不一致**：正文叙述的均值、标准差、最小值、最大值、样本量等，与对应描述性统计表中的数字不一致。（例：正文写“均值0.266、标准差0.089”，但表格为“0.269、0.086”。）
2. **回归系数或显著性不一致**：正文叙述的系数值、正负号、显著性水平，与对应回归表不一致。
3. **结论方向与回归表矛盾**：正文/摘要/结论对分组或异质性结果的大小排序、正负方向、强弱梯度的文字描述，与回归表实际系数不一致。（例：文字写“中部＞西部＞东部”，但表中西部系数最大。）
4. **中英文摘要关键信息不一致**：研究对象、样本期间、地区范围、核心方法、主要结论方向、政策时期等关键信息在中英文摘要之间不一致，或英文摘要内部自相矛盾。（例：中文“十五五”，英文写“14th Five-Year Plan”。）
5. **核心数据口径前后不一致**：样本期间、样本量、地区范围、核心变量口径在摘要、数据说明、表格、结论之间明显冲突。
6. **正文引用与参考文献严重不匹配**：正文明确引用的文献在参考文献中缺失，或编号体系明显错乱到无法对应（仅个别格式瑕疵不在此列，属黄色）。
7. **语言强度完全超出实证证据范围**：摘要、副标题、结论前段使用统计学专有词（显著、双重、证明、因果）或具体异质性比较词（更明显、更突出、更强、排序为 X＞Y＞Z），但实证章节对应维度完全未做回归（非则仅是缺少部分控制变量、稳健性或细化分组）。不得仅因作者在结论章末尾提供了“理论推导”免责声明而降为黄色；免责声明必须在摘要就同步展现才能视为对齐。（例：摘要写“存在显著双重负向抑制作用”但实证只做了一个被解释变量；摘要写异质性排序但实证章未做分组回归。）

判级要求：

- 上述任一情形一经确认，`level` 必须为“红色”，`evidence_strength` 必须为“强”，且 evidence 字段必须**同时列出相互矛盾的两处证据**（例如“正文段落#73:0.266”与“表格#3:0.269”）。
- 只有当相关内容位于无法解析的图片/公式对象、无法确认两处数值时，才可转为灰色需人工确认；一旦两处数值均可读且确实矛盾，不得转灰、不得降级。
- 判断“矛盾”以可解析文本为准，不得臆测；但发现可读的数字冲突时，必须判红，不得回避。

## 9ter. 视觉证据强度规则（v1.6+ · M3 视觉模块）

当 `student_evidence.source_type` 为 `"vision"` 或 `"vision_failed"` 时，适用本节。本节不改变 § 9 / § 9bis 对“两处独立可读、两处矛盾”的定义，仅确定视觉证据进入两处时的强度与判级允许。

### 9ter.1 视觉证据的三个关键字段

- `quality_profile.extraction_quality_score` ∈ [0, 1]：提取质量分（结构完整度 + 识别覆盖度）
- `quality_profile.critical_field_score` ∈ [0, 1]：关键字段完整度（表格的变量列/系数列/显著性、公式的变量符号等）
- `quality_profile.hard_gates_passed` ∈ {true, false}：硬门禁校验是否通过（存在 cells / 存在关键行 / 无敏感词命中等）

三个字段均由 `scripts/vision/quality_scorer.py` 系统结构校验打分，**不采信** Provider 自报的 confidence。

### 9ter.2 判级门槛

| 证据组合 | 视觉证据条件 | 允许判级 |
|---|---|---|
| 两处独立可读，**至少一处 DOCX / PDF 原生文本** | vision 端需 `hard_gates_passed=true` + `extraction_score ≥ 0.75` | 红 · 黄 · 绿 · 灰 |
| 一处 DOCX / PDF 文本 + 一处 vision | vision 端需 `hard_gates_passed=true` + `extraction_score ≥ 0.75` | **黄** · 绿 · 灰（不得判红） |
| 两处均为 vision | 任意 | 绿 · **灰**（不得判红与黄） |
| 单处 vision 作提醒 | 任意 | **灰** · need_manual_confirmation=true |
| `source_type="vision_failed"` 或 `hard_gates_passed=false` | / | **灰** + 进 `manual_confirmation_items` |

**M3.1 硬约束**：`pure_vision_evidence_can_be_red = false`。即即使视觉识别质量极高，只要证据全部来自视觉，**不自动判红**。红色判决必须有至少一处 DOCX 原生文本或 PDF 文本层证据支撑。

### 9ter.3 视觉证据的 evidence 字段写法

- 视觉提取的 `parsed_text` **可以**直接展示在 `issue.evidence` 字段（作为学生论文内容的一部分）
- 视觉证据需带前缀：`[视觉解析·请核对原图]`
- **不得**在 evidence 正文硬编码告警语（如“本内容视觉识别可能错”）；告警只能通过 `review_notice` 字段承载，由渲染器展示
- 视觉证据对应的 issue 必须同时写入 `vision_evidence` 字段（含 `page_number` / `thumbnail_ref` / `quality_profile` / `kb_eligibility` / `table_preview`），供报告渲染展示缩略图与预览表

### 9ter.4 KB 门禁（与 evidence 门槛一致）

走 `scripts/vision/knowledge_bridge.py::compute_kb_eligibility()`：

- `kb_a_query_eligible`：所有视觉结果均可用于 KB-A **查询**
- `kb_a_evidence_eligible`：`hard_gates_passed = true` + `extraction_score ≥ 0.75` 方可作为 issue 依据
- `kb_b_eligible`：`hard_gates_passed = true` + `extraction_score ≥ 0.85` 方可挂参照卡

不合格的视觉结果仍可挂灰色 issue 作人工核对提醒，但**不得**作为红黄绿判定依据。

### 9ter.5 视觉证据与三层证据分离

| 三层 | 视觉可否 |
|---|---|
| `student_evidence` | ✅ 可（`source_type="vision"`） |
| `normative_basis`（KB-A） | ❌ 视觉永远不作规范依据内容 |
| `example_reference`（KB-B） | ❌ 视觉永远不作参照卡内容 |

详见 `agent_instructions/vision_protocol.md`。

## 10. 需人工确认触发条件

表格为图片或复杂嵌入对象、模型公式无法解析、图表注释无法稳定识别、参考文献格式解析不完整、论文类型无法稳定判断、数据类型无法稳定判断、是否需要高级模型属于导师判断范围、学校格式模板未提供、原文信息可能位于附录脚注图片中、证据不足但存在潜在风险时，必须进入需人工确认。

## 11. 负证据处理规则

论文未检测到某项内容属于负证据。可以作为问题的情况包括本科经管论文通常必须呈现的基础信息未检测到、当前论文已使用某方法但未检测到该方法必要说明、论文前文声称做了某项分析但后文未检测到对应结果、摘要正文表格结论之间存在明显缺口。

不得作为问题的负证据包括论文未使用 DID、IV、PSM，案例论文未设置回归模型，课程论文未设置完整实证结构，调查数据库论文未做自编问卷信度效度检验。

## 12. 表格、公式、图片证据边界

若表格内容可读取，可以作为证据。若表格为图片、截图或复杂嵌入对象，不能直接判定表格缺少某项内容，应输出“回归表可能为图片或复杂嵌入对象，系统无法稳定识别变量名、系数、标准误和显著性星号，建议人工确认”。

若模型公式不可解析，不得直接判断模型方程错误，应输出“模型公式无法稳定解析，需人工确认变量符号、下标和固定效应项说明是否完整”。

## 13. 证据强度与等级矩阵

强证据可输出红色、黄色或绿色。中证据可输出黄色、绿色或灰色。弱证据优先输出灰色或黄色。无证据不得输出任何 issue。

## 14. 输出前证据自检

生成最终报告前逐条检查：每个问题是否有 evidence，每个问题是否有 location，evidence 是否来自用户上传论文，evidence 是否足以支撑 issue_type，红色问题是否达到强证据门槛，弱证据是否已转为需人工确认，是否存在编造变量模型或数据，是否存在把未检测到写成学生没有做，是否存在不适用规则被写成问题，是否存在高级模型强制要求。

## 15. KB-B 范例层引用红线（v1.4+）

KB-B 是本 Skill 内置的顶刊范例论文向量库，仅供改进参照，**不得作为学生论文的错误判定依据**。写入报告前必须确认：

- KB-B 命中片段不能出现在 issue 的 evidence 字段中（evidence 只允许来自用户上传论文）；
- KB-B 命中片段不能作为「因为范例这么写所以学生那么写不对」的推论根据；
- KB-B 命中片段只能出现在 `改进参照` 字段（参见 issue_writing_protocol.md 第 4bis 节）；
- KB-B 命中片段不得写入 red 级别问题的 evidence；即使写入 yellow/green，也不得皮以 example 弱化真实原文证据。

层次区分：
- **student_evidence**（学生证据）：只能来自学生上传的论文 units；
- **normative_basis**（规范依据）：只能来自 KB-A 规范层、学校模板、国标、权威教材、Skill 已审核规则库；
- **example_reference**（范例参照）：来自 KB-B（顶刊已脱敏论文向量库），仅作改进方向参照。

三层不得互相代替。内部优先级：student_evidence → normative_basis → example_reference。
