# 对外声明边界（M4 · Public Claims Policy）

**约束级别**：硬红线
**上游**：`plans/M4_BUILD_GATE_ADDENDUM.md` §7

---

## 一、每次公开公示必须同时给出

- `dataset_version`
- 完整论文样本数量 + 小型夹具数量
- 覆盖学科（经管本科为主）+ 论文类型（DOCX / 文本 PDF / 扫描 PDF / 图表密集 / KB 稀疏）
- **未覆盖场景**（如硕博盲审、期刊终稿、跨学科等）
- Skill 版本 + 规则版本 + KB 版本 + 视觉模型版本 + endpoint id
- 测试日期 + 测试环境（OS / Python / dependency_lock_hash）
- Mock vs 真调 Ark 的区分标签

---

## 二、禁止表述（黑名单）

以下表述不得出现在任何公开材料、README、CHANGELOG、Scorecard、PPT、答辩稿、社交媒体中：

- ❌ "适用于所有学科和所有论文"
- ❌ "整体准确率 XX%"（首版样本量不足以代表任何总体）
- ❌ "可替代教师 / 导师 / 专家审阅"
- ❌ "所有 PDF / 表格 / 公式均能正确识别"
- ❌ "视觉模型识别结果绝对可信"
- ❌ "通过 N 篇论文即可证明系统普适有效"
- ❌ "由 AI 全权判定，无需人工复核"

## 三、推荐官方措辞

### 3.1 中性描述模板

> 当前版本 v1.7.0 已在指定的 DOCX（1 篇）、文本型 PDF（1 篇）、扫描 PDF（1 篇）、图表密集论文（1 篇）、KB-B 稀疏场景（1 篇）中完成回归验证。测试结果仅代表当前数据集（`benchmark-dataset-v0.1`）、模型（火山方舟 doubao-vision-v0.1）、规则（rules-2026.07）与运行环境下的表现，不能替代教师或专业评审。

### 3.2 用于火山杯答辩的谦逊版

> 我们不宣称这个工具能通用地判断所有论文。我们做的是：**给每一版 Skill 一份可复核的成绩单**，让老师、评委、开发者都能知道"这一版在什么场景下有把握、在什么场景下会降级到人工复核"。

## 四、Scorecard 数字披露规则

### 4.1 允许的量化描述

- 硬门禁字段全 = 0 的样本数 / 总样本数（例 5/5）
- p50 / p95 运行耗时（**只报本样本集**，不外推）
- 参考锚点 anchor_recall（**只报本样本集**）
- KB-A / KB-B 命中数（**只报本样本集**）

### 4.2 禁止的量化描述

- "红判准确率 95%"（首版 5 样本远不足以支持这类统计）
- "视觉表格识别 100%"（除非视觉子领域样本 ≥ 30 且各类交叉）
- "召回率 / 精确率 / F1"（除非同时公示样本量与置信区间）

### 4.3 与"回归验证"的区别

- **回归验证**：这一版没比上一版差 → 允许公开
- **绝对准确率**：这一版对不对 → **不允许公开**（需要更大样本 + 独立评审）
