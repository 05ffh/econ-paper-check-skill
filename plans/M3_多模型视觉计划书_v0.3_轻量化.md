## 4. 新架构：单流程 · 单 Provider

```
              ┌───────────────┐              ┌───────────────────┐
              │  .docx 输入   │              │  .pdf 输入        │
              │  （推荐）     │              │                   │
              └───────┬───────┘              └────────┬──────────┘
                      │                                │
                      │                                ▼
                      │                       ┌────────────────┐
                      │                       │ PDF 页级分诊    │
                      │                       │ + 礼貌提醒      │
                      │                       └────────┬───────┘
                      │                                │
                      │                    ┌───────────┼───────────┐
                      │                    ▼           ▼           ▼
                      │              [文本页 OK]  [图片区域]  [扫描页]
                      │                    │           │           │
                      │                    │           ▼           ▼
                      │                    │     ┌─────────────────────┐
                      │                    │     │ 视觉辅助 Provider   │
                      │                    │     │ （Ark 云 · 单实例）  │
                      │                    │     └──────────┬──────────┘
                      │                    │                │
                      │                    │           ┌────┴────┐
                      │                    │           ▼         ▼
                      │                    │      [已配置]   [未配置]
                      │                    │           │         │
                      │                    │           ▼         ▼
                      │                    │     cells[]+bbox   ⬛ 降灰
                      │                    │                    +原图缩略
                      ▼                    ▼        ▼           ▼
             ┌──────────────────────────────────────────────────────┐
             │       判断内核 · LLM 语义判断（M1 完整保留）          │
             │       rules/ + agent_instructions/ + references/      │
             └──────────────────────────┬───────────────────────────┘
                                        ▼
             ┌──────────────────────────────────────────────────────┐
             │       M1 三层证据 · KB-A 规范 + KB-B 参照            │
             │       student_evidence（视觉证据经 knowledge_bridge）  │
             └──────────────────────────┬───────────────────────────┘
                                        ▼
                             ┌────────────────────┐
                             │  报告渲染          │
                             │  .docx + .html     │
                             └────────────────────┘
```

**关键变化**：
- **无 mode_resolver**：视觉 Provider `available()` 决定"启用/未启用"
- **无自动降级**：可用就用，不可用就降灰
- **视觉未配置 ≠ 报错**：直接判"灰"（无法判定），学生看到"该图片/表格无法自动识别，请人工审阅"
- **视觉已配置**：Ark 云 Provider 处理，走 knowledge_bridge 挂 M1

## 6. 环境变量简化（12 个 → 4 个）

| v0.3 环境变量 | 默认 | 说明 |
|---|---|---|
| `VISION_ENABLED` | `false` | 用户显式开关 |
| `ARK_API_KEY` | *(未设)* | 火山方舟 Key |
| `ARK_VISION_MODEL_ID` | *(未设)* | 视觉模型 ID（部署者配，**不绑豆包**） |
| `ARK_BASE_URL` | `https://ark.cn-beijing.volces.com/api/v3` | Ark 端点 |

**删除**：`RUNTIME_MODE` / `LOCAL_VISION_ENABLED` / `LOCAL_MODEL_PATH` / `LOCAL_DEVICE` / `ALLOW_CLOUD_UPLOAD` / `ALLOW_FULL_PAGE_UPLOAD` / `DEBUG_KEEP_RAW_RESPONSE` 等 8 个。

**启用条件**（三合一）：`VISION_ENABLED=true` **+** `ARK_API_KEY` 有效 **+** `ARK_VISION_MODEL_ID` 已设。任何一个缺失 → 视觉不启用 → 图片内容降灰。

**上传前用户确认**：即便三条件全满足，第一次上传含图片的 PDF 时，仍向用户提示"该 PDF 含图片区域，是否允许上传到火山方舟识别"（一次性授权，会话内记住）。

## 9. Provider 契约不变（Phase 2 已定）

单一 Ark 云 Provider 完整实现，契约仍按 `scripts/vision/providers/base.py`：

- **幂等**：同 image_hash + 同任务 → 同结果哈希
- **无副作用**：不改传入文件
- **失败**：raise ProviderError（NOT_CONFIGURED / NETWORK / TIMEOUT / QUOTA / UNAUTHORIZED / INTERNAL）
- **不做质量评分**：由 quality_scorer 系统校验
- **不做隐私裁剪**：由 privacy_gate 前置（隐私模块保留）
- **不做缓存**：内嵌简单 image_hash → response 内存 LRU

**接口保持 OpenAI Compatible**：走 `ARK_BASE_URL` + `ARK_VISION_MODEL_ID`，**不绑豆包**，部署者配任何 Ark 视觉多模态模型都能用。

**支持任务类型**（v0.3 单 Provider 全覆盖）：
- OCR_FULL_PAGE / OCR_REGION（整页 / 区域 OCR）
- TABLE_STRUCTURE（表格结构 → cells[]）
- LAYOUT（版面分析 · 用于定位图/表/公式区域）
- REFERENCE_BLOCK（参考文献块）
- FORMULA（M3.1 不作强制 DoD，能识别更好，不能识别降灰）

## 10. 图片降灰的具体产物

**视觉未启用** 或 **Provider 失败** 或 **quality_score < 阈值** 时：

```yaml
issue:
  level: gray
  type: figures_tables
  location: "第 13 页 · 图 1"
  student_evidence:
    source_type: "pdf_image_region"    # 明确是图片
    parsed_text: null                   # 无法识别
    page_number: 13
    bbox: [x0, y0, x1, y1]
    artifact_ref: "/path/to/张弘济初稿.pdf#p13"
    thumbnail_ref: ".workdir/.vision_render/张弘济初稿_p13_thumb.png"
  reason: "该区域为图片/图表，未启用视觉辅助识别"
  suggestion: "请人工审阅原图；如需自动识别，请联系部署者配置视觉辅助能力"
  requires_visual_review: true
  review_notice: "视觉未启用 · 请核对原图"
```

**报告层**：灰色 issue 附上**原图缩略图**（PDF 页渲染缩略，走 pypdfium2 或 pdfplumber），学生一看就知道 AI 没法评这块，需要自己核对。

**保底**：即便完全没配视觉，学生依然能拿到完整的**文本层质检报告 + 图片区域的灰色提示 + 原图缩略**。

**注意**：缩略图渲染要求 `pypdfium2` 或 `pdfplumber` 已装 —— `pdfplumber` 在 core 依赖中，缩略图能力**默认可用**。

## 16. 用户可见的最终形态
