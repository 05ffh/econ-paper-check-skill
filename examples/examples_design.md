# 经管论文智检 Skill：示例论文设计规范

## 1. 文件作用

本文件规定 examples 目录中示例论文的设计要求。当前版本不生成真实示例 DOCX，仅保留设计规范。

第一版建议后续至少包含 sample_paper_clean.docx 和 sample_paper_with_errors.docx。前者用于测试 Skill 是否会克制判断、输出通过项和不适用项；后者用于测试 Skill 是否能识别典型本科经管论文问题。

## 2. 示例论文总体原则

示例论文应使用 .docx 格式，属于经管类本科论文或课程论文，不涉及真实学生个人隐私，不包含真实学号、身份证号、手机号等敏感信息，不直接使用真实未授权论文全文，可以使用脱敏、改写或模拟内容，问题应贴近真实本科论文痛点，不刻意制造过于夸张的问题，能测试论文画像、规则触发、证据门禁和报告生成。

## 3. sample_paper_clean.docx 设计要求

该文件应是一篇相对规范的经管类本科实证论文样本。它不要求完美，但应具备本科论文基本结构。

建议主题包括数字金融发展对居民消费水平的影响研究，或绿色金融对企业创新投入的影响研究。

建议结构包括题目、中文摘要、英文摘要、关键词、目录、引言、文献综述、理论分析、数据来源与变量定义、模型设定、描述性统计、基准回归、稳健性检验、结论与建议、参考文献。

应具备的规范点包括题目能体现研究对象和核心变量，中文摘要包含研究对象数据方法结论，英文摘要与中文摘要基本一致，文献综述有分类，数据来源说明较清楚，变量定义表较完整，模型公式说明基本完整，回归表包含 N、R² 和显著性说明，结论能对应研究结果，参考文献与正文引用基本一致。

可保留少量轻微问题，例如政策建议略偏泛化，图表注释可以更完整，英文关键词个别术语可统一，文献述评可以更集中。这些问题不应大量触发红色问题。

预期检测结果是总体风险低风险或中低风险，红色必改问题 0 到 1 项，黄色建议补充 2 到 5 项，绿色优化提升 2 到 5 项，需人工确认 0 到 2 项，通过项不少于 5 项，不适用项包含 DID、IV、问卷信度效度等未触发说明。

## 4. sample_paper_with_errors.docx 设计要求

该文件应是一篇包含典型本科经管论文问题的样本。

建议主题为数字经济对区域共同富裕水平的影响研究。这个主题适合测试面板数据、综合评价或熵值法、固定效应模型、中介机制、结论与政策建议、中英文摘要一致性。

建议结构包括题目、中文摘要、英文摘要、关键词、引言、文献综述、指标体系与数据来源、模型设定、实证结果、机制分析、结论与政策建议、参考文献。

## 5. error 样本必须命中清单

必须命中以下问题：

- 研究问题不明确，对应 TQ-QUESTION-001；
- 文献综述罗列化，对应 LT-REVIEW-001；
- 数据来源不可追溯，对应 DV-SOURCE-001；
- 核心变量定义不完整，对应 DV-VAR-001；
- 熵值法说明不足，对应 DV-INDEX-001 或 MM-ENT-001；
- 模型方程说明不完整，对应 MM-GEN-002；
- 强因果表述，对应 MM-GEN-003；
- 回归表注释不完整，对应 MM-GEN-004 或 SN-TABLE-001；
- 中英文摘要关键信息不一致，对应 SN-ABSTRACT-EN-001；
- 正文引用与参考文献不一致，对应 SN-REF-001；
- 政策建议套话化，对应 SN-ADVICE-001。

如果上述核心问题多数未命中，说明规则执行不足，需要回到规则库或语义判断协议修正。

## 6. clean 样本不得误报清单

clean 样本用于检查 Skill 是否过度挑错。对该样本，不得因为未使用 DID 就提示缺少 DID，不得因为未使用 IV 就提示缺少工具变量，不得因为未使用 PSM 就提示稳健性不足，不得把轻微表达问题升级为红色，不得把已有但较简略的文献述评直接判为严重问题，不得把本科论文没有复杂识别策略判为模型错误，不得输出大量低价值语言问题，不得编造论文中不存在的变量、表格或结论。

理想结果是通过项较多、问题项较少、风险等级较低、不适用项清楚。

## 7. 示例论文隐私与合规要求

示例论文不得包含真实学生姓名、真实学号、真实导师姓名、真实班级、真实联系方式、未授权真实论文全文、真实论文中的敏感数据、明显可追溯到个人的写作痕迹。

可以使用模拟姓名、模拟学号、模拟数据片段、改写后的论文段落、公开常识性主题、虚构但合理的表格。

## 8. 示例论文与验收用例映射

sample_paper_clean.docx 可测试论文画像生成、条件触发、证据门禁和 DOCX 报告完整性。

sample_paper_with_errors.docx 可测试论文画像生成、条件触发、中英文摘要一致性、强因果表述和 DOCX 报告完整性。

可选案例论文样本可测试案例论文克制。可选调查数据库样本可测试调查数据库识别。可选图片表格样本可测试需人工确认。

## 9. 示例文件缺失时处理方式

如果 Skill 包中暂时没有真实 sample_paper_clean.docx 或 sample_paper_with_errors.docx，不影响 Skill 主体使用。用户仍可上传自己的 DOCX 论文使用本 Skill。

比赛展示前，建议补齐至少两个示例 DOCX：一个相对规范样本，一个典型问题样本。若条件允许，再补充一个案例分析样本和一个调查数据库样本。

## 10. 示例论文质量门禁

示例论文进入正式 Skill 包前，应检查是否为 .docx，是否已脱敏，是否不包含真实学生隐私，是否能被 Claude Code 读取，是否能生成论文画像，是否能触发预期规则，是否不会误触发不适用规则，是否能生成 DOCX 报告，是否适合比赛演示，是否能说明 Skill 的真实价值。
